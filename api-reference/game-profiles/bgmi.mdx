Stan lets users link their Battlegrounds Mobile India (BGMI) accounts to their profiles. Once linked, the BGMI in-game account ID is exposed through the Stan user object, giving you a verified bridge between a player's Stan identity and their BGMI identity.

## Where the BGMI Profile ID Lives

The BGMI account ID is surfaced in two places:

- **User object** — the `bgmiProfileId` field returned by `GET /api/v1/users`.
- **JWT token payload** — the same integer is embedded in the authenticated user's token, so your server can read it without an extra API round-trip.

`bgmiProfileId` is a plain integer that maps directly to the player's unique BGMI in-game ID. No additional transformation is needed before passing it to BGMI's own tooling.

## Retrieving the BGMI Profile ID

Call `GET /api/v1/users` with the player's Stan username to fetch their profile. The `bgmiProfileId` field in the response contains the linked BGMI account ID.

```bash
curl --http2 -X GET \
  -H "Host: api.getstan.app" \
  -H "accept: application/json, text/plain, */*" \
  -H "gaid: 9bbb0f02-5c2c-4a3b-8083-1fa26eeaedf4" \
  -H "appversion: 253" \
  -H "stallionversion: 0" \
  -H "platform: android" \
  -H "sid: 1784387335639-45684" \
  -H "ts: 1784387581188" \
  -H "appinstanceid: 76238ef97361a56425efb91beb0fdd57" \
  -H "installreferrer: utm_source=google-play&utm_medium=organic" \
  -H "countrycode: in" \
  -H "fingerprintid: 1784387336407.dba6a7" \
  -H "authorization: Bearer <YOUR_TOKEN>" \
  -H "accept-encoding: gzip" \
  -H "user-agent: okhttp/4.12.0" \
  "https://api.getstan.app/api/v1/users?username=ItzDakshbro1"
```

The response highlights the `bgmiProfileId` field:

```json
{
  "id": 32091378,
  "username": "ItzDakshbro1",
  "displayName": "Daksh",
  "bgmiProfileId": 59704785,
  "freefireProfileId": 59704786,
  "countryCode": "in",
  "avatarUrl": "https://cdn.getstan.app/avatars/32091378.jpg"
}
```

In this example, `59704785` is the player's BGMI in-game ID.

## When bgmiProfileId is null

If a Stan user has not linked a BGMI account, `bgmiProfileId` returns `null`:

```json
{
  "id": 32091378,
  "username": "ItzDakshbro1",
  "displayName": "Daksh",
  "bgmiProfileId": null,
  "freefireProfileId": 59704786,
  "countryCode": "in",
  "avatarUrl": "https://cdn.getstan.app/avatars/32091378.jpg"
}
```

Always check for `null` before attempting to use the value downstream. You should prompt the user to link their BGMI account from within the Stan app if you need this ID and it is absent.

## Use Cases

- **In-game identity verification** — confirm that a Stan user is the genuine owner of a specific BGMI account before awarding prizes or granting tournament access.
- **Stat display** — pass the `bgmiProfileId` to BGMI's own APIs or a third-party stat tracker to fetch match history, K/D ratio, rank, and other in-game statistics.

<Note>
  Stan does not expose a direct BGMI stats API. Use the `bgmiProfileId` value with BGMI's official APIs or third-party stat-tracking services (such as BGMIStats or similar platforms) to retrieve in-game performance data.
</Note>
