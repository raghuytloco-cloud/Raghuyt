The Stan API uses Bearer JWT authentication. Every request you make must carry a signed token in the `Authorization` header. The API validates the token on each call — no session cookies, no API keys. This page explains what the token contains, how long it lasts, and exactly how to pass it.

## Authentication Scheme

Stan uses the standard HTTP Bearer scheme. Place your JWT directly after the `Bearer` keyword, separated by a single space.

<ParamField header="authorization" type="string" required>

## Token Structure

Stan JWTs are signed with the **HS256** algorithm. Once decoded, the payload contains the following claims:

<ResponseField name="id" type="number" required>
  The unique numeric identifier of the authenticated Stan user (e.g. `32091378`). Use this to correlate API responses back to a specific account.
</ResponseField>

<ResponseField name="appType" type="string" required>
  Always `"stan"` for valid Stan tokens. If you receive a token where this field has any other value, treat it as invalid.
</ResponseField>

<ResponseField name="bgmiProfileId" type="number">
  The user's linked Battlegrounds Mobile India (BGMI) profile ID, if connected (e.g. `59704785`).
</ResponseField>

<ResponseField name="freefireProfileId" type="number">
  The user's linked Free Fire profile ID, if connected (e.g. `59704786`).
</ResponseField>

<ResponseField name="iat" type="number" required>
  Issued-at timestamp as a Unix epoch (seconds). Indicates when the token was created.
</ResponseField>

<ResponseField name="exp" type="number" required>
  Expiry timestamp as a Unix epoch (seconds). The API rejects any token presented after this time with a `401` error.
</ResponseField>

### Decoded Payload Example

```json
{
  "appType": "stan",
  "bgmiProfileId": 59704785,
  "exp": 1785434935,
  "freefireProfileId": 59704786,
  "iat": 1782842935,
  "id": 32091378
}
```

## Token Lifetime

Stan tokens are valid for approximately **30 days** from the `iat` (issued-at) timestamp. In the example above, the token was issued on `2025-04-30` and expires on `2025-05-29`.

<Note>
  You can calculate the remaining lifetime at any time with `exp - Math.floor(Date.now() / 1000)` (JavaScript) or `exp - int(time.time())` (Python). A negative result means the token has already expired.
</Note>

## Obtaining a Token

Stan tokens are issued through the **Stan mobile app** (Android / iOS). Authenticate inside the app to receive a token, then extract it from the app's network traffic for use in your API calls.

<Warning>
  Never share your JWT or commit it to source control. Anyone who holds your token can act on your behalf until it expires.
</Warning>

## Passing the Token

Include the token in the `authorization` header of every request alongside all other required platform-identity headers. The header is case-insensitive, but the `Bearer ` prefix (note the trailing space) is required.

```bash cURL
curl --http2 -X GET \
  -H "Host: api.getstan.app" \
  -H "accept: application/json, text/plain, */*" \
  -H "gaid: 9bbb0f02-5c2c-4a3b-8083-1fa26eeaedf4" \
  -H "appversion: 253" \
  -H "stallionversion: 0" \
  -H "platform: android" \
  -H "sid: 1784387335639-45684" \
  -H "ts: 1784387581188" \
  -H "appinstanceid: 76238ef97361a56425efb91beb0fdd57" \
  -H "installreferrer: utm_source=google-play&utm_medium=organic" \
  -H "countrycode: in" \
  -H "fingerprintid: 1784387336407.dba6a7" \
  -H "authorization: Bearer <YOUR_TOKEN>" \
  -H "accept-encoding: gzip" \
  -H "user-agent: okhttp/4.12.0" \
  "https://api.getstan.app/api/v1/users?username=ItzDakshbro1"
```

## Authentication Errors

When authentication fails, the API returns a `401 Unauthorized` response. The three most common causes are a missing token, an expired token, and a malformed token.

```json
{
  "statusCode": 401,
  "message": "Unauthorized",
  "error": "Unauthorized"
}
```

| Scenario                         | HTTP Status | How to resolve                                              |
|----------------------------------|-------------|-------------------------------------------------------------|
| No `authorization` header sent   | `401`       | Add the `authorization: Bearer <token>` header.             |
| Token has passed its `exp` time  | `401`       | Re-authenticate via the Stan app to obtain a fresh token.   |
| Token is malformed or corrupted  | `401`       | Verify the full token string was copied without truncation. |

<Tip>
  Check the `exp` claim of your token before sending requests in long-running scripts. Proactively refreshing the token before it expires avoids mid-session `401` failures.
</Tip>
