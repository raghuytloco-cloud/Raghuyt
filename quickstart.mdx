This guide gets you from zero to a live API response as fast as possible. You'll obtain a Bearer token from the Stan mobile app, assemble the required request headers, and call the user-lookup endpoint to fetch a public gamer profile. The whole process takes about five minutes.

<Steps>
  <Step title="Get your Bearer token">
    Stan uses JWT Bearer tokens for authentication. Tokens are issued by the Stan mobile app — there is no web-based login or OAuth flow.

    1. Install the **Stan** app on an Android device (available on the Google Play Store).
    2. Sign up or log in with your account.
    3. Capture the `Authorization` header value from any outgoing API request using a traffic-inspection tool such as [HTTP Toolkit](https://httptoolkit.com) or a proxy like Charles or mitmproxy.

    The token looks like this:

    ```text
    Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
    ```

    <Note>
      Your token is valid for approximately 30 days. Store it securely — it grants access to your account's public API scope. See the [Authentication guide](/authentication) for full details on the token structure and expiry behaviour.
    </Note>
  </Step>

  <Step title="Assemble the required headers">
    Every Stan API request requires a specific set of headers beyond just `Authorization`. These headers describe the client environment and are validated server-side. Assemble them as follows:

    | Header | Example value | Description |
    |---|---|---|
    | `authorization` | `Bearer <YOUR_TOKEN>` | Your JWT Bearer token |
    | `platform` | `android` | Always set to `android` |
    | `appversion` | `253` | Stan app build version |
    | `sid` | `1784387335639-45684` | Session ID — `<timestamp>-<random>` |
    | `ts` | `1784387581188` | Current Unix timestamp in milliseconds |
    | `gaid` | `9bbb0f02-5c2c-4a3b-8083-1fa26eeaedf4` | Google Advertising ID (UUID) |
    | `appinstanceid` | `76238ef97361a56425efb91beb0fdd57` | Unique device installation ID (hex) |
    | `fingerprintid` | `1784387336407.dba6a7` | Device fingerprint |
    | `countrycode` | `in` | ISO 3166-1 alpha-2 country code |
    | `stallionversion` | `0` | Internal version field — always `0` |
    | `installreferrer` | `utm_source=google-play&utm_medium=organic` | Install attribution string |

    <Tip>
      For `ts`, generate the value fresh on each request using the current epoch time in milliseconds — for example, `Date.now()` in JavaScript or `int(time.time() * 1000)` in Python. Stale timestamps may cause requests to be rejected.
    </Tip>
  </Step>

  <Step title="Make your first API call">
    With your token and headers ready, call the user-lookup endpoint. Pass the Stan username you want to look up as the `username` query parameter.

    <CodeGroup>
      ```bash cURL
      curl --http2 -X GET \
        -H "Host: api.getstan.app" \
        -H "accept: application/json, text/plain, */*" \
        -H "gaid: 9bbb0f02-5c2c-4a3b-8083-1fa26eeaedf4" \
        -H "appversion: 253" \
        -H "stallionversion: 0" \
        -H "platform: android" \
        -H "sid: 1784387335639-45684" \
        -H "ts: 1784387581188" \
        -H "appinstanceid: 76238ef97361a56425efb91beb0fdd57" \
        -H "installreferrer: utm_source=google-play&utm_medium=organic" \
        -H "countrycode: in" \
        -H "fingerprintid: 1784387336407.dba6a7" \
        -H "authorization: Bearer <YOUR_TOKEN>" \
        -H "accept-encoding: gzip" \
        -H "user-agent: okhttp/4.12.0" \
        "https://api.getstan.app/api/v1/users?username=ItzDakshbro1"
      ```

      ```python Python
      import requests
      import time

      url = "https://api.getstan.app/api/v1/users"
      params = {"username": "ItzDakshbro1"}

      headers = {
          "accept": "application/json, text/plain, */*",
          "gaid": "9bbb0f02-5c2c-4a3b-8083-1fa26eeaedf4",
          "appversion": "253",
          "stallionversion": "0",
          "platform": "android",
          "sid": "1784387335639-45684",
          "ts": str(int(time.time() * 1000)),
          "appinstanceid": "76238ef97361a56425efb91beb0fdd57",
          "installreferrer": "utm_source=google-play&utm_medium=organic",
          "countrycode": "in",
          "fingerprintid": "1784387336407.dba6a7",
          "authorization": "Bearer <YOUR_TOKEN>",
          "user-agent": "okhttp/4.12.0",
      }

      response = requests.get(url, headers=headers, params=params)
      print(response.json())
      ```

      ```javascript JavaScript
      const ts = Date.now().toString();

      const response = await fetch(
        "https://api.getstan.app/api/v1/users?username=ItzDakshbro1",
        {

            appversion: "253",
            stallionversion: "0",
            platform: "android",
            sid: "1784387335639-45684",
            ts: ts,
            appinstanceid: "76238ef97361a56425efb91beb0fdd57",
            installreferrer: "utm_source=google-play&utm_medium=organic",
            countrycode: "in",
            fingerprintid: "1784387336407.dba6a7",
            authorization: "Bearer <YOUR_TOKEN>",
          },
        }
      );

      const data = await response.json();
      console.log(data);
      ```
    </CodeGroup>
  </Step>

  <Step title="Read the response">
    A successful request returns a `200 OK` with a JSON body containing the user's public profile. Here's what the response looks like:

    ```json
    {
      "id": 32091378,
      "username": "ItzDakshbro1",
      "displayName": "Daksh",
      "bgmiProfileId": 59704785,
      "freefireProfileId": 59704786,
      "countryCode": "in",
      "avatarUrl": "https://cdn.getstan.app/avatars/32091378.jpg"
    }
    ```

    <ResponseField name="id" type="number">
      The Stan user's unique numeric identifier.
    </ResponseField>
    <ResponseField name="username" type="string">
      The user's unique Stan username (case-sensitive).
    </ResponseField>
    <ResponseField name="displayName" type="string">
      The user's chosen display name, which may differ from their username.
    </ResponseField>
    <ResponseField name="bgmiProfileId" type="number">
      The numeric profile ID of the user's linked BGMI account, if connected.
    </ResponseField>
    <ResponseField name="freefireProfileId" type="number">
      The numeric profile ID of the user's linked Free Fire account, if connected.
    </ResponseField>
    <ResponseField name="countryCode" type="string">
      ISO 3166-1 alpha-2 country code for the user's registered region.
    </ResponseField>
    <ResponseField name="avatarUrl" type="string">
      Absolute URL to the user's profile avatar image on Stan's CDN.
    </ResponseField>

    If your token is missing or expired, the API returns a `401` error instead:

    ```json
    {
      "statusCode": 401,
      "message": "Unauthorized",
      "error": "Unauthorized"
    }
    ```

    <Warning>
      If you see a `401 Unauthorized` response, your token has likely expired. Log back into the Stan app to capture a fresh token and retry your request.
    </Warning>
  </Step>
</Steps>
